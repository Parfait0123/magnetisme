01 — Authentification

API Key

🔑

Header requis sur tous les endpoints de prédiction

Chaque requête de prédiction doit inclure le header X-API-Key avec la clé fournie. Le endpoint /health est public.

Header 
X-API-Key: Test123

En cas d'absence → 401 Unauthorized  ·  Clé incorrecte → 403 Forbidden

POST /api/v1/predict/binary Prédiction binaire — upload 🔒 Auth

Classifie une image histologique en Benign (bénin) ou Malignant (malin). Corps de la requête en multipart/form-data.

Paramètres

ChampTypeStatutDescription
file File Requis Image JPEG, PNG, TIFF ou BMP · max 10 MB
cURL
curl -X POST \
  https://parfait60-breast-cancer-classification-api.hf.space/api/v1/predict/binary \
  -H "X-API-Key: votre-clé-api" \
  -F "file=@/chemin/image.png"
JavaScript
const formData = new FormData();
formData.append('file', imageFile); // imageFile = objet File du <input>

const response = await fetch(
  'https://parfait60-breast-cancer-classification-api.hf.space/api/v1/predict/binary',
  {
    method: 'POST',
    headers: { 'X-API-Key': 'votre-clé-api' },
    body: formData,
  }
);
const result = await response.json();

console.log(result.prediction);  // "Malignant"
console.log(result.confidence);  // 0.987432
Python
import requests

url = "https://parfait60-breast-cancer-classification-api.hf.space/api/v1/predict/binary"
headers = {"X-API-Key": "votre-clé-api"}

with open("image.png", "rb") as f:
    response = requests.post(url, headers=headers, files={"file": f})

result = response.json()
print(result["prediction"])   # "Malignant"
print(result["confidence"])   # 0.987432

  "probabilities": {
    "Benign": 0.012568,

  }
}
POST /api/v1/predict/binary/base64 Prédiction binaire — Base64 🔒 Auth

Même prédiction binaire mais l'image est envoyée en JSON encodé Base64. Utile quand l'image est déjà en mémoire côté client.

Body JSON

ChampTypeStatutDescription
image string Requis Image en Base64 · raw ou avec préfixe data:image/...;base64,
JavaScript
// Convertir un File en base64
const toBase64 = (file) => new Promise((resolve) => {
  const reader = new FileReader();
  reader.onload = (e) => resolve(e.target.result);
  reader.readAsDataURL(file);
});

const base64 = await toBase64(imageFile);

const response = await fetch(
  'https://parfait60-breast-cancer-classification-api.hf.space/api/v1/predict/binary/base64',
  {
    method: 'POST',
    headers: {
      'X-API-Key': 'votre-clé-api',
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({ image: base64 }),
  }
);
Python
import base64, requests

with open("image.png", "rb") as f:
    b64 = base64.b64encode(f.read()).decode()

response = requests.post(
    "https://parfait60-breast-cancer-classification-api.hf.space/api/v1/predict/binary/base64",
    headers={"X-API-Key": "votre-clé-api"},
    json={"image": b64}
)
POST /api/v1/predict/multiclass Classification 8 classes — upload 🔒 Auth

Classifie l'image parmi 8 types histologiques du dataset BreaKHis. Retourne les probabilités pour chaque classe et un top 3.

JavaScript
const formData = new FormData();
formData.append('file', imageFile);

const response = await fetch(
  'https://parfait60-breast-cancer-classification-api.hf.space/api/v1/predict/multiclass',
  {
    method: 'POST',
    headers: { 'X-API-Key': 'votre-clé-api' },
    body: formData,
  }
);
const result = await response.json();

console.log(result.prediction);    // "Fibroadenoma"
console.log(result.top3);          // [{label, confidence}, ...]
Python
import requests

url = "https://parfait60-breast-cancer-classification-api.hf.space/api/v1/predict/multiclass"
with open("image.png", "rb") as f:
    response = requests.post(url,
        headers={"X-API-Key": "votre-clé-api"},
        files={"file": f}
    )
result = response.json()
print(result["top3"])

POST /api/v1/predict/multiclass/base64 Classification 8 classes — Base64 🔒 Auth

Version Base64 du endpoint multiclasse. Body JSON identique à /predict/binary/base64 — même format { "image": "..." }. Réponse identique à /predict/multiclass.

ℹ Même logique que /predict/binary/base64 — seul le endpoint change. Remplacer /binary/ par /multiclass/ dans l'URL.

04 — Gestion des erreurs

Codes HTTP

200

Success

Prédiction effectuée avec succès.

401

Unauthorized

Header X-API-Key manquant dans la requête.

403

Forbidden

Valeur de l'API Key incorrecte.

413

Too Large

Image trop volumineuse. Limite : 10 MB.

415

Unsupported Media

Format non accepté. Utiliser JPEG, PNG, TIFF ou BMP.

422

Unprocessable

Image corrompue ou Base64 invalide.

429

Too Many Requests

Limite de 60 requêtes/minute par IP dépassée.

500

Internal Error

Erreur interne du serveur.

Format d'erreur 
{
  "detail": "Description de l'erreur",
  "code": "INTERNAL_ERROR"
}
